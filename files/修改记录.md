## 二、头文件 `filesys.h` 的修改

### 2.1 结构体精确对齐（跨平台修复）
**问题**：原始代码在 16 位 Turbo C 下开发，`unsigned int` 为 2 字节；在 32/64 位系统上为 4 字节，导致 `dinode`(32B)、`direct`(16B) 等结构体大小全部错乱，磁盘布局崩溃。

**修复**：
- 引入 `<stdint.h>`，使用 `uint16_t`、`uint32_t` 等定宽类型
- 对所有结构体（包括内存结构体和磁盘结构体）统一使用 `#pragma pack(push, 1)` / `#pragma pack(pop)` 禁止编译器自动填充
- 确保 `sizeof(struct dinode) == 32`、`sizeof(struct direct) == 16`、`sizeof(struct filsys)` 一致

### 2.2 统一 `di_size` 类型
**问题**：`struct inode` 中 `di_size` 为 `unsigned short`(2B)，`struct dinode` 中为 `unsigned long`(4B)，读写时不对齐。

**修复**：统一为 `uint32_t`。

### 2.3 修正拼写错误
- `UDIEXICUTE` → `UDIEXECUTE`
- `GDIEXICUTE` → `GDIEXECUTE`
- `ODIEXICUTE` → `ODIEXECUTE`
- `EXICUTE` → `EXECUTE`
- 删除 `#define SEEK_SET 0`（与标准 C 冲突）

### 2.4 函数声明补全
**问题**：原始代码大量 `extern iput();`、`extern bfree();` 等缺少返回类型和参数类型，默认按 `int` 处理，导致类型不匹配。

**修复**：全部补齐为 ANSI C 风格声明，例如：
```c
extern void iput(struct inode *pinode);
extern void bfree(uint16_t block_num);
extern uint16_t namei(const char *name);
```

### 2.5 新增函数声明
- `void rmdir(const char *dirname);`
- `void cp(uint16_t user_id, const char *src, const char *dst);`

## 四（补）. 基于 `filesystem1` 对比审查的二次修复

> 以下修复通过对比 `filesystem1` 修复版本发现，当前项目仍存在同样问题，已逐一修复。所有修复**不删除、不回退**任何已有扩展功能。

### 4.1 权限位定义改为标准 UNIX 八进制（`filesys.h`）
**问题**：原权限宏位定义与标准 UNIX 相反（`UDIREAD=00001` 在 bit0，标准应在 bit8），导致 `chmod 0755` 后 `dir` 显示的 `rwxr-xr-x` 与实际权限位完全不对应。
**修复**：将权限宏调整为标准位：
- 用户：`UDIREAD=00400`、`UDIWRITE=00200`、`UDIEXECUTE=00100`
- 组：`GDIREAD=00040`、`GDIWRITE=00020`、`GDIEXECUTE=00010`
- 其他：`ODIREAD=00004`、`ODIWRITE=00002`、`ODIEXECUTE=00001`

### 4.2 `open.c` — 禁止打开目录
**问题**：`open workdir read` 会把目录当成普通文件打开。
**修复**：`aopen()` 中 `iget` 后立即检查 `inode->di_mode & DIDIR`，若是目录则拒绝。

### 4.3 `creat.c` — 禁止覆盖目录 + 权限参数生效
**问题**：
1. `create workdir` 会尝试覆盖已有目录；
2. 新建文件时实际使用 `u_default_mode`，没有使用命令传入的 `mode`；
3. `sys_ofile[i].f_flag = mode` 把文件权限（如 `01777`）错当成打开模式。
**修复**：
1. 覆盖前增加 `inode->di_mode & DIDIR` 检查；
2. 新建时改为 `inode->di_mode = (mode & 0777) | DIFILE`；
3. `f_flag` 统一设为 `FWRITE`。

### 4.4 `dir.c` + `main.c` — `cd` 失败不更新路径
**问题**：`cd missing` 失败后，提示符路径已被错误更新，造成显示路径和真实目录不一致。
**修复**：
1. `chdir()` 返回值从 `void` 改为 `int`（成功返回 1，失败返回 0）；
2. `main.c` 中仅在 `chdir(args)` 返回非 0 时才更新 `current_path`。

### 4.5 `igetput.c` — 释放 inode 时标记空闲
**问题**：`iput()` 在 `di_number == 0` 时只释放数据块和调用 `ifree()`，没有将磁盘 inode 的 `di_mode` 写回 `DIEMPTY`，后续扫描空闲 inode 时状态不一致。
**修复**：释放后设置 `pinode->di_mode = DIEMPTY`、`pinode->di_size = 0`，并将空闲状态写回磁盘 inode 区，再调用 `ifree()`。

### 4.6 `dir.c` — `mkdir` 磁盘满时回滚
**问题**：`balloc()` 失败后直接返回，已分配的目录项和 inode 未清理。
**修复**：失败时清除目录项、恢复 `dir.size`、将 inode 引用数置零并 `iput()` 释放。

## 五、新增拓展功能

### 5.1 `cp`（文件复制）
**位置**：`main.c` 中 `cmd_cp()` 函数  
**实现**：打开源文件（只读）→ 创建目标文件 → 循环 `read`/`write` 直到文件结束 → 关闭双方。完全复用已有系统调用。

### 5.2 `rmdir`（删除空目录）
**位置**：`dir.c` 中新增 `rmdir()` 函数  
**实现**：检查目录是否存在 → 检查是否为目录 → 读取目录数据块检查是否只有 `.` 和 `..` → 释放数据块 → `iput` → 从父目录移除。

### 5.3 `chmod`（修改文件权限）
**位置**：新增 `chmod.c`  
**实现**：查找文件 inode → 检查是否为所有者 → 保留高位的文件类型标志，修改低 9 位权限位 → `iput` 写回。

### 5.4 `create` 带内容（创建时直接写入）
**位置**：`main.c` 中 `cmd_create()`  
**实现**：解析命令行时，若文件名后还有额外字符串，在 `creat()` 成功后立即 `write()` 写入。支持可选八进制模式参数，如 `create file 0755 hello`。

### 5.5 硬链接 / 软链接
**位置**：新增 `link.c`，修改 `open.c`、`delete.c`  
**实现**：
- **硬链接 `hlink`**：在目录中新建项指向已有 inode，`di_number++`。`delete()` 已支持引用计数，减到 0 才释放块。
- **软链接 `slink`**：创建 `DILINK` 类型文件，内容为目标路径。`aopen()` 中增加软链接解析，递归打开目标文件。

### 5.6 多级索引（一次间址）
**位置**：`rdwt.c` 核心重构，影响 `igetput.c`、`creat.c`、`delete.c`、`dir.c`  
**实现**：
- `addr[0]~addr[8]`：9 个直接索引块（4608B）
- `addr[9]`：一次间址块（指向一个数据块，存储 256 个块号 = 128KB）
- 新增 `get_block()` / `set_block()` / `free_all_blocks()` 统一封装块寻址与释放
- 文件最大容量从 5KB 提升到约 **132KB**（4608B + 128×512B）
- `_dir()` 已支持打印间接块中的块号链

## 七、已知限制（课设范围内可接受）

1. **目录项稀疏存储**：目录中删除文件后会留下空洞（`d_ino = 0`），`dir.size` 只减不增，长期大量增删可能浪费少量目录槽位。

## 八、Bug 修复与鲁棒性增强（第二轮）

### 8.1 `halt.c` — 目录块泄漏修复
**问题**：`halt()` 写回当前目录时，对每个目录块都直接 `balloc()` 分配新块，导致旧数据块被重复泄漏；目录缩小时也不释放多余块。
**修复**：
- 计算 `old_nblocks`（基于 `di_size`）与 `new_nblocks`（基于当前 `dir.size`）
- 复用 `di_addr[]` 中已有的非零块号，仅当块号为 0（且不是根目录的 `.` 块）时才 `balloc()`
- 目录缩小时，对 `new_nblocks ~ old_nblocks` 范围内的块调用 `bfree()` 释放

### 8.2 `mv.c` — 跨目录移动修复
**问题**：`mv file subdir/name` 时，目标目录项被错误地写入了当前目录，而不是 `subdir`。
**修复**：
- 新增 `load_dir_entries()` / `write_dir_entries()`：完整地读取/写回目标目录的所有数据块，同样支持块复用与缩容
- 新增 `valid_name()`：校验文件名非空、不含 `/`、长度合法
- 跨目录分支：解析 `parent/name`，验证 `parent` 是子目录且有写权限，将 inode 插入目标目录，再从当前目录清零源项

### 8.3 `format.c` / `main.c` — 运行时格式化状态过期修复
**问题**：在已挂载的文件系统上执行 `format` 后，旧 `fd` 未关闭、内存中 `hinode[]` 哈希链残留旧 inode、`cur_path_inode` 悬空、`user_id` 和 `current_path` 仍指向旧状态，导致后续操作崩溃或访问旧数据。
**修复**：
- `format.c`：`format()` 开头关闭旧 `fd`（若已打开），并调用新增 `clear_inode_cache()` 遍历释放 `hinode[]` 中所有链表节点
- `main.c`：`cmd_format()` 在 `format(); install();` 后重置 `user_id = -1` 和 `current_path = "/"`

### 8.4 `access.c` — 标准 UNIX 权限语义修复
**问题**：原始实现允许 owner 身份被 group/other 权限“兜底”；root 用户也没有特殊处理。
**修复**：
- 采用严格的 owner → group → other 优先级：一旦匹配身份，只检查对应权限位，不向后 fallback
- root（`uid == 0`）直接放行所有访问
- 验证场景：`chmod 0050` 时 owner 无法打开（即使 group 允许）、`chmod 0004` 时 other 可以打开

### 8.5 `superblock.c/h` — 超级块模块化
**问题**：超级块初始化逻辑散落在 `format.c`、`install.c`、`halt.c` 中。
**修复**：
- 新增 `superblock.c` / `superblock.h`
- 提供 `sb_init_format()`（格式化时初始化成组链接）、`sb_read()` / `sb_write()`（读写 block 1）、`sb_sync()`（按需写回）
- `format.c`、`install.c`、`halt.c` 全部调用这些 API，消除重复代码

### 8.6 UI 美化（ANSI 颜色）
**位置**：`filesys.h` 新增宏，`main.c` / `dir.c` / `stat.c` / `wc.c` / `df.c` 等应用  
**实现**：
- 定义 `C_BOLD`、`C_RED`/`GREEN`/`BLUE`/`CYAN`/`YELLOW`、`C_DIM`、`C_RESET` 及高亮变体
- 提示符 `[uid:path]$` 使用蓝色，成功消息绿色，错误消息红色，`dir` 中目录/链接分别使用青色/青色高亮，`stat` 标签加粗，`wc` / `df` 数字高亮

### 8.7 `test.sh` — 边界测试增强
- **权限边界**：`chmod 0000` / `0050` / `0004`，跨用户（2118/2119/2117）切换验证严格匹配
- **FD 表耗尽**：连续创建 20 个文件（填满 `NOFILE=20`），第 21 个返回“User open file table full”
- **非空目录删除**：在目录内创建文件后从父目录 `rmdir`，确认被拒绝
- **间接块读取**：`lseek 4608`（块 9，间接索引第 1 项）读取 `JJJJ...`，验证多级索引正确性
